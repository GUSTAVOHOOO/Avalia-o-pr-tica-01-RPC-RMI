  <action>
    Em game_server.py, dentro do método start_game(), localizar a criação do TurnMachine (por volta da linha 485). Atualmente o código passa:

      on_round_start=lambda: self._assign_images_for_turn(room_code_for_cb),
      on_hint_phase_start=lambda: self._consume_image_assignments(room_code_for_cb),

    Antes da construção do TurnMachine, definir uma função local com return explícito:

      def _hint_phase_cb():
          self._assign_images_for_turn(room_code_for_cb)
          return self._consume_image_assignments(room_code_for_cb)

    Então passar on_hint_phase_start=_hint_phase_cb no construtor do TurnMachine.

    O on_round_start pode ser removido (se TurnMachine aceita None) ou mantido como lambda: None. Verificar a assinatura de TurnMachine.__init__ em turn_machine.py para saber se o parâmetro é opcional antes de remover.

    IMPORTANTE — formas proibidas:
    - Não usar lambda tuple: `lambda: (assign(), consume())` retorna uma tupla (None, dict), não o dict. Em turn_machine.py linha 166, `dict(image_assignments or {})` recebe a tupla — como ela é truthy, `or {}` é ignorado e `dict((None, actual_dict))` lança TypeError.
    - Não usar função local sem return: Python retorna None implicitamente; `dict(None or {})` resulta em {} e image_assignments fica sempre vazio.
    - A ÚNICA forma correta é a função local com `return` explícito mostrada acima.

    A mudança deve ser cirúrgica: apenas o bloco de criação do TurnMachine em start_game() é afetado. Nenhum outro método é alterado.
