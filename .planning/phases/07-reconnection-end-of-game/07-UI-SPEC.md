### Chat visual separation (D-04 / CHAT-03 / CHAT-04)

Chat panel uses Secondary background (#1a1d27) with a left border in `#6b7280` (muted gray) — distinct from game-phase panel left borders (blue `#3b82f6`, green `#22c55e`, yellow `#eab308`). The chat submit button label is "Enviar mensagem" and uses accent (#6366f1) only for its background. Game-action inputs (hint, guess) retain their blue/green phase-accent left borders.

## Interaction States

### Vote timer bar

- Fill: accent (#6366f1) from 30s → 11s
- Fill: warning (#eab308) from 10s → 6s
- Fill: destructive (#ef4444) from 5s → 0s
- Animation: CSS `transition: width 1s linear` per second tick

Pattern source: same three-state timer color progression established by REQUIREMENTS.md UI-05, which is being implemented in Phase 8 for the general phase timer. The vote timer uses an identical color progression for consistency.

### Podium cards

- 1st place card: top border 3px solid #22c55e, padding-top 32px (elevated vs. 2nd/3rd at 16px)
- 2nd/3rd place cards: no colored border, standard secondary bg card

### Chat messages scroll

- New message appended to bottom; container scrolls to bottom automatically on append
- If user has scrolled up manually, do not auto-scroll (preserve reading position)

### Reconnect flow (client-side)

On page load (GameScreen.tsx `useEffect`):
1. Read `player_id` + `room_code` from localStorage
2. If both exist, emit `reconnect_game` event instead of `join_room`
3. While waiting for ack: show inline text "Reconectando…" in `.game-screen__status`
4. On success ack: hide status, apply restored state, render game normally
5. On error ack: show ".game-screen__status--error" with "Sessão expirada. Entre na partida novamente." and a link back to landing page
