<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser localStorage → Socket.IO → Bridge | player_id read from localStorage; arbitrary string; validated server-side in reconnect_player() |
| Bridge timer thread → Bridge SID dicts | do_leave() closure accesses _pending_leaves under lock; concurrent reconnect can race with timer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Spoofing | handle_reconnect_game() client-supplied player_id | mitigate | server-side reconnect_player() validates UUID against _player_to_room before re-registering; bridge trusts server response |
| T-07-03-02 | Tampering | _pending_leaves race (Pitfall 3) | mitigate | handle_reconnect_game() cancels timer AND removes dict entry under _pending_leaves_lock before calling reconnect_player(); atomic cancel prevents do_leave() from running after reconnect |
| T-07-03-03 | Tampering | send_chat player_id spoofing | mitigate | SID → player_id lookup under _sid_lock; only the authenticated SID can send as that player_id |
| T-07-03-04 | Spoofing | Timer keyed by player_id instead of SID | mitigate | _pending_leaves keyed by SID per Pitfall 1 in RESEARCH.md; search by player_id only for cancellation, not keying |
</threat_model>
