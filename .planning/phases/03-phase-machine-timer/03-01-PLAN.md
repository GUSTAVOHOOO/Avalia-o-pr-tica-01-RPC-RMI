    Create server/turn_machine.py with a TurnMachine class. Key implementation details:

    Module-level constant: PHASE_SEQUENCE = ["ROUND_START", "HINT_PHASE", "GUESS_PHASE", "EXCHANGE_PHASE", "SPY_PHASE", "SCORING_PHASE", "TURN_END"]

    TurnMachine.__init__(self, room_code: str, max_turns: int, broadcaster):
      - self.room_code, self.max_turns, self.broadcaster stored as instance attributes
      - self.lock = threading.RLock()
      - self.current_phase: str = "WAITING"
      - self.current_turn: int = 1
      - self._generation: int = 0
      - self._timer_handle: Optional[threading.Timer] = None
      - self._phase_start_time: float = 0.0

    TurnMachine.start(self):
      Calls self._advance_to("ROUND_START")
      Called by GameServer.start_game() after GAME_STARTED broadcast (per RESEARCH.md open question 2).

    TurnMachine._advance_to(self, phase: str, from_timer: bool = False, expected_generation: int = -1):
      CRITICAL ORDERING — must follow this exact pattern to avoid broadcast-inside-lock deadlock:
        1. Enter with self.lock block
        2. If from_timer and self._generation != expected_generation: log stale timer (room, expected, current gen), return — exit lock before any broadcast
        3. Cancel existing self._timer_handle if not None; set to None
        4. Increment self._generation; capture gen_snapshot = self._generation
        5. Set self.current_phase = phase; self._phase_start_time = time.monotonic()
        6. If phase == "GAME_ENDED": set game_ended = True, build broadcast_data without "phase" key (contains room_code, current_turn, max_turns)
        7. Else: build broadcast_data dict with keys: room_code, phase, remaining_seconds (from config.PHASE_DURATIONS.get(phase, 30)), current_turn, max_turns
              Create timer callback closure: captures gen_snapshot (NOT self._generation), calls self._compute_next(phase) then self._advance_to(next_phase, from_timer=True, expected_generation=gen_snapshot)
              self._timer_handle = threading.Timer(duration, callback); self._timer_handle.daemon = True; self._timer_handle.start()
        8. Exit with self.lock block
        9. AFTER lock exits: if game_ended call self.broadcaster.broadcast("game_ended", broadcast_data), else call self.broadcaster.broadcast("phase_changed", broadcast_data)

    TurnMachine._compute_next(self, current_phase: str) -> str:
      Acquires self.lock (RLock re-entry is safe since same thread always calls this while holding lock from _advance_to, but _compute_next itself also acquires for safety).
      If current_phase == "TURN_END":
        if self.current_turn >= self.max_turns: return "GAME_ENDED"
        self.current_turn += 1; return "HINT_PHASE"  (D-06: skip ROUND_START on subsequent turns)
      Else: idx = PHASE_SEQUENCE.index(current_phase); return PHASE_SEQUENCE[idx + 1]

    TurnMachine.advance_phase_manual(self):
      Compute next_phase under lock (to read current_phase safely), then call self._advance_to(next_phase).
      Do NOT pass from_timer=True — this is a manual advance, not a timer callback.

    TurnMachine.remaining_seconds property:
      Acquires self.lock; computes max(0, int(config.PHASE_DURATIONS.get(self.current_phase, 0) - (time.monotonic() - self._phase_start_time)))

    Imports needed: threading, time, logging, typing.Optional, config
    No Pyro5 import — TurnMachine is a pure Python class (per RESEARCH.md primary recommendation).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Timer thread → TurnMachine state | threading.Timer fires in its own thread; must acquire RLock before reading or writing any TurnMachine field |
| TurnMachine → EventBroadcaster | Network I/O crosses this boundary; must not be called while holding the RLock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | TurnMachine._advance_to — double-advance via stale timer | mitigate | Generation counter (TURN-04): timer callback checks self._generation == expected_generation before acting; mismatch → log + return. Tested in test_generation_counter. |
| T-03-02 | Denial of Service | TurnMachine._advance_to — deadlock from broadcasting inside lock | mitigate | Broadcast-outside-lock pattern: all broadcaster.broadcast() calls happen after the with self.lock block exits. Matches existing game_server.py join_game/start_game pattern. |
| T-03-03 | Elevation of Privilege | TurnMachine._compute_next — current_turn mutation race | mitigate | _compute_next always called from _advance_to which holds self.lock (RLock); mutation of current_turn is atomic within the lock. |
| T-03-04 | Denial of Service | threading.Timer handle — double-cancel on fired timer | accept | threading.Timer.cancel() is safe to call on already-fired timer (stdlib no-op). Handle set to None after cancel to prevent confusing double-cancel logic. Low risk. |
| T-03-05 | Elevation of Privilege | config.PHASE_DURATIONS — test monkeypatching corrupts production state | accept | Monkeypatching only in test scope with try/finally restore. No production code path touched. Acceptable for test utilities. |
</threat_model>
