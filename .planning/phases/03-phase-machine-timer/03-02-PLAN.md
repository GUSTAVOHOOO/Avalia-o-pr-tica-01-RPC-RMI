<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pyro5 timer thread → GameServer.advance_phase() | Timer thread calls TurnMachine methods; GameServer.advance_phase() called by Pyro5 handler thread; both must acquire TurnMachine.lock before touching phase state |
| BridgeCallbackReceiver Pyro5 daemon thread → socketio.emit() | on_phase_changed fires in bridge's Pyro5 background thread; socketio.emit() must be thread-safe (confirmed for threading mode) |
| TurnMachine timer thread → GameServer._set_session_ended() | on_game_ended callback fires from TurnMachine's timer thread; _set_session_ended() acquires self.lock independently — no re-entrancy risk as TurnMachine does not hold GameServer.lock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Elevation of Privilege | advance_phase() — unauthorized caller can skip phases | accept | Authorization deferred per CONTEXT.md — Phase 3 scopes this as test/operator utility only. No player-facing UI exposes this method in Phase 3. |
| T-03-07 | Denial of Service | start_game() — turn_machine.start() called before game_started broadcast arrives at bridge | mitigate | Enforce call order: broadcaster.broadcast("game_started", ...) executes first; turn_machine.start() called after. Lobby.tsx navigates to /game/:roomCode on game_started; 5s ROUND_START provides buffer for mount (Pitfall 4 in RESEARCH.md). |
| T-03-08 | Denial of Service | BridgeCallbackReceiver.on_phase_changed — socketio.emit() throws if room is empty | accept | Wrapped in try/except; error is logged but does not crash the bridge thread. Empty room is a no-op (no subscribers). |
| T-03-09 | Spoofing | advance_phase() — no player_id validation, fallback to first IN_PROGRESS session | accept | Phase 3 explicitly defers authorization (CONTEXT.md). The fallback is a test convenience that Phase 4+ removes when authorization is tightened. |
| T-03-10 | Denial of Service | _set_session_ended() — deadlock if TurnMachine holds GameServer.lock when callback fires | mitigate | TurnMachine never acquires GameServer.lock; _set_session_ended() acquires GameServer.lock independently. No nested lock acquisition path exists. |
