## Screen Layout Contract — GameScreen (`/game/:roomCode`)

Phase 3 renders a single-zone layout. The full 4-zone layout (UI.md WEB-006) is built in Phase 4.

```
[PhaseHeader]
  [PhaseBadge: phase name + color]   [CountdownDisplay: N seconds]   [TurnIndicator: "Turno X de Y"]

[BodyArea — full remaining height]
  [Centered placeholder text]
  "Aguardando ação do servidor..."
```

Max content width: 640px centered. Background: `#0f1117`.

PhaseHeader layout: flex row, space-between, padding `md` (16px) horizontal, `sm` (8px) vertical. Background: `#1a1d27`. Border-bottom: 1px solid `#2d3148`.

Turn indicator: 14px/400, color `#6b7280`. Format: "Turno 1 de 5".

Placeholder body area: centered vertically and horizontally. Text: 16px/400, color `#6b7280`.

### Socket.IO Event Wiring

| Event | Handler | UI Effect |
|-------|---------|-----------|
| `phase_changed` | Update `currentPhase` + `remainingSeconds` state | PhaseBadge re-renders with new color; CountdownDisplay resets to new value |
| `game_ended` | Set `gameEnded = true` | Replace placeholder with "Jogo encerrado" message; clear countdown |

Countdown ticking: client-side `setInterval(1000)` decrements `remainingSeconds` locally. Resets to server value on each `phase_changed` event. Stops at 0 (does not go negative).

## Interaction Contracts

### GameScreen (`/game/:roomCode`)

- On mount: join Socket.IO room via `socket.emit('join_room', { room_code: roomCode })`
- Listen for `phase_changed` event: `{ phase: string, remaining_seconds: number, current_turn: number, max_turns: number }`
- Listen for `game_ended` event: `{ room_code: string }` — sets ended state, stops countdown
- Countdown tick: `setInterval` at 1000ms, decrements local `remainingSeconds` by 1, clamps at 0
- On each `phase_changed` event: clear existing interval, reset seconds to `remaining_seconds`, start new interval
- Component unmount: clear interval, remove socket listeners
- `roomCode` read from `useParams()` — React Router route param
- `playerId` read from `localStorage.getItem('player_id')` — written in Phase 2

### Navigation

- Lobby navigates to `/game/:roomCode` on `game_started` event — already wired in Phase 2 `Lobby.tsx`
- Phase 3 does not add any navigate-away logic from the game screen (Phase 7 handles results redirect)
