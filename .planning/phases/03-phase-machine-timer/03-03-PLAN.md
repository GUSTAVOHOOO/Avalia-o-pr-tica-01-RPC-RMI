<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (client) → Bridge Socket.IO | join_room event from untrusted browser; room_code is user-supplied |
| Bridge → Browser | phase_changed/game_ended events include room_code from server state (trusted) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Information Disclosure | GameScreen — roomCode read from URL param; anyone with URL joins the Socket.IO room | accept | Phase 3 game screen is display-only (no actions). Joining the room only receives phase_changed events. No sensitive data exposed. Authentication deferred to later phase. |
| T-03-11 | Spoofing | GameScreen — playerId from localStorage could be tampered | accept | Phase 3 does not use playerId for any server-side action from GameScreen. localStorage.getItem is read-only in this component. Risk is Phase 4+ when actions are added. |
| T-03-12 | Denial of Service | CountdownDisplay — setInterval memory leak if component unmounts without cleanup | mitigate | cleanup function in useEffect clears intervalRef.current on unmount and removes socket listeners. Tested as part of smoke test (navigate away and back). |
| T-03-13 | Tampering | App.tsx route /game/:roomCode — path traversal via malformed roomCode | accept | React Router v7 handles route matching; roomCode is passed to socket.emit and display only; no filesystem access. Low risk for this read-only component. |
</threat_model>
