<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Bridge (Socket.IO) | player_name, room_code, max_turns arrive as browser-supplied strings/ints; bridge must not trust them blindly |
| Bridge → GameServer (Pyro5) | bridge forwards browser payload; GameServer performs authoritative validation under lock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
