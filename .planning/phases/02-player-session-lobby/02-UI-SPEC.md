## Color

The game theme is espionagem/blefe/traição — dark, moody, atmospheric. All screens use a dark base.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1117` | Page background, main surface — dark near-black |
| Secondary (30%) | `#1a1d27` | Cards, form containers, lobby player list panel, nav header |
| Accent (10%) | `#6366f1` | Primary CTA buttons, active input borders, focus rings, host badge |
| Destructive | `#ef4444` | Leave lobby confirmation button, inline error messages only |

Accent (`#6366f1`, indigo-500) reserved ONLY for:
- "Criar Partida" primary CTA button (WEB-001, WEB-002)
- "Iniciar Jogo" button when enabled (WEB-005, host only)
- "Entrar" primary submit button (WEB-003/004)
- Active/focused input border ring
- Host badge background in player list

All other interactive elements use the secondary surface color with a white/muted text contrast.

Additional semantic colors (not part of the 60/30/10 base — used contextually only):
- Success: `#22c55e` (green-500) — "Jogador entrou" toast, copy-link confirmation
- Warning: `#f59e0b` (amber-500) — reserved for Phase 3+ timer states (declared here for consistency)
- Destructive: `#ef4444` (red-500) — inline errors, leave confirmation (listed above)
- Muted text: `#6b7280` (gray-500) — placeholder text, secondary descriptors
- Primary text: `#f1f5f9` (slate-100) — all body and heading copy on dark surfaces

## Interaction Contracts

### WEB-001 Landing Page

- Room code input normalizes to uppercase on every keystroke (`onChange: value.toUpperCase().slice(0, 6)`)
- "Entrar" on landing is secondary — visually smaller than "Criar Partida"
- No validation on landing code field until submit; validation happens on navigate to `/join/:code`

### WEB-002 Criar Partida

- Nickname: maxLength=20, pattern=`[A-Za-z0-9 _-]+`, validated on submit (not onBlur)
- Turn count: `<select>` with options [3, 5, 7, 10]; default selected value: 5
- Submit button shows spinner + disables all fields while `create_game` RPC in flight
- On success: navigate to `/lobby/:sessionId`; `player_id` stored in `localStorage.player_id`

### WEB-003/004 Entrar na Partida

- WEB-003: Room code input auto-uppercases; validated onBlur (inline loading spinner while checking)
- WEB-004: Room code pre-filled from URL param, disabled input; only nickname required
- After valid code check: show preview "X jogadores aguardando" below code field
- Submit button disabled until both code (valid) and nickname (non-empty) are filled

### WEB-005 Lobby

- Player list updates in real time via `PLAYER_JOINED` Socket.IO event — new player animates in (fade + slide-down, 200ms)
- Player departure: item fades out (150ms) + toast "Jogador X saiu do lobby"
- Host leaving: toast "Novo host: [nome]" appears for all remaining players
- "Iniciar Jogo" button: enabled only when `players.length >= 2` AND `currentPlayer.is_host === true`
- "Iniciar Jogo" while loading: spinner replaces button text, all inputs disabled, status message "Distribuindo imagens..."
- Room code display: monospace font (`font-mono`), 28px, centered; copy button uses `navigator.clipboard.writeText`
- Copy fallback: if Clipboard API unavailable, select-all text and show "Selecione e copie manualmente"
- `localStorage.player_id` must be written before navigating to lobby

## Screen Layout Contracts

### Landing (`/`)

```
[Header: Logo + "Regras" link]
[Hero: Título + tagline + ilustração]
[Cards x3: mecânicas]
[CTAs: "Criar Partida" (primary, full-width) | "Entrar" field + button (secondary)]
[Footer: link regras]
```

Max content width: 640px centered. Full viewport height hero on desktop.

### Criar Partida (`/create`)

```
[Back link "← Voltar"]
[Title "Nova Partida" — heading 20px/600]
[Form:
  Label "Apelido" + TextInput + InlineError
  Label "Número de turnos" + TurnSelect + helper "~3 min por turno"
  PrimaryButton "Criar"
]
```

Form container: max-width 400px, centered, secondary bg card, lg padding (24px).

### Entrar na Partida (`/join` and `/join/:code`)

```
[Back link "← Voltar"]
[Title "Entrar na Partida"]
[Form:
  (WEB-003 only) Label "Código da Sessão" + TextInput[maxLength=6, auto-uppercase] + InlineError
  (WEB-004) disabled pre-filled code field
  [Session preview: "X jogadores aguardando" — shown after code validated]
  Label "Apelido" + TextInput + InlineError
  PrimaryButton "Entrar"
]
```

### Lobby (`/lobby/:sessionId`)

```
[Header: "Sala de Espera" + room code badge]
[RoomCodeDisplay + CopyLinkButton]
[PlayerCountIndicator]
[PlayerList: PlayerListItem × N (live updates)]
[Footer:
  (host) PrimaryButton "Iniciar Jogo" [disabled if <2 players]
  SecondaryButton "Sair do Lobby"
]
```

## Accessibility Baselines

- All interactive elements: min 44px touch target
- Focus ring: 2px solid `#6366f1` with 2px offset on all focusable elements
- Color is never the sole indicator of state — all errors have text labels
- Form inputs have explicit `<label>` association (htmlFor / id)
- Disabled states use `opacity-50` + `cursor-not-allowed`
- Room code display uses `aria-label="Código da sala: ABC123"`
