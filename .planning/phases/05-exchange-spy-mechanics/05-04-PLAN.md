Private delivery pattern (on_object_assigned — exact model for on_exchange_requested, on_exchange_hints, on_spy_success):
```python
@Pyro5.api.oneway
@Pyro5.api.callback
def on_object_assigned(self, data: dict):
    target_player_id = data.get("target_player_id")
    with _sid_lock:
        sid = _player_to_sid.get(target_player_id)
    if not sid:
        print(f"[BRIDGE] player SID not found for {target_player_id}", flush=True)
        return
    try:
        socketio.emit("object_assigned", data, to=sid)
        print(f"[BRIDGE] object_assigned -> sid={sid} player={target_player_id}", flush=True)
    except Exception as exc:
        print(f"[BRIDGE] ERROR in on_object_assigned: {exc}", flush=True)
```

@Pyro5.api.oneway

def on_hint_received(self, data: dict):

    except Exception as exc:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Bridge (Socket.IO) | Untrusted client input enters here; player identity resolved from SID map, never from payload |
| Bridge → GameServer (Pyro5 RPC) | Bridge passes validated player_id from SID map; exchange_id and other params from payload are validated by GameServer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Spoofing | handle_request_exchange player identity | mitigate | player_id always resolved from `_sid_to_player.get(request.sid)` inside _sid_lock; target_player_id in payload is validated server-side against session.players set |
| T-05-10 | Information Disclosure | on_exchange_hints private routing | mitigate | socketio.emit("exchange_hints", data, to=sid) sends to individual SID only; not to=room_code; SID is opaque to other clients |
| T-05-11 | Information Disclosure | on_spy_success private routing | mitigate | socketio.emit("spy_success", data, to=sid) — spy success hints reach spy's SID only; no room broadcast (SPY-03) |
| T-05-12 | Spoofing | handle_attempt_spy exchange_id source | accept | exchange_id is validated server-side: checked against turn_state.completed_exchanges, then record ownership checked for self-spy; bridge passes it through without validation — low risk since server is authoritative |
</threat_model>
