<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Bridge → GameServer | player_id resolved server-side from SID map in bridge; client payload never used for identity |
| Client payload → exchange_id | exchange_id validated against turn_state.exchanges and turn_state.completed_exchanges inside the lock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02 | Spoofing | request_exchange() target_player_id | mitigate | Bridge resolves caller from _sid_to_player map; target_player_id in payload validated against session.players set inside lock — cannot forge identity |
| T-05-03 | Repudiation | exchange_id across turns | accept | TurnState (including exchanges dict) is reset on each HINT_PHASE; old exchange_ids are simply absent from turn_state.exchanges; no explicit replay protection needed |
| T-05-04 | Tampering | submit_exchange_hint before acceptance | mitigate | `record.status != "accepted"` guard returns {"error": "exchange_not_accepted"} — prevents submitting hints on pending or rejected exchanges (Pitfall 6) |
| T-05-05 | Elevation of Privilege | attempt_spy() repeated calls | mitigate | spy_attempts set blocks second attempt per turn; probability resolved server-side only; client cannot influence random.random() result |
| T-05-06 | Information Disclosure | EXCHANGE_COMPLETED payload | mitigate | broadcast_data contains only {room_code, exchange_id, requester_id, target_id} — no hint words; private hints sent via send_to_player only (EXCHANGE-04, EXCHANGE-05) |
| T-05-07 | Elevation of Privilege | Spy on own exchange | mitigate | `if player_id in (record.requester_id, record.target_id)` guard returns {"error": "cannot_spy_own_exchange"} (SPY-04) |
| T-05-08 | Tampering | Spy on non-completed exchange | mitigate | Guard checks `exchange_id not in turn_state.completed_exchanges` (not turn_state.exchanges) — prevents spying on rejected/pending exchanges (D-02, Pitfall 4) |
</threat_model>
