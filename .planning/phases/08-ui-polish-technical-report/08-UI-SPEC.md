## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, chip inner vertical padding |
| sm | 8px | Field gap, button row gap, card inner gap, hint chip horizontal gap |
| md | 16px | Default card padding, section gap, header padding, content gap |
| lg | 24px | Page section gap, bottom toast positioning |
| xl | 32px | Page body padding, podium first-place top padding |
| 2xl | 48px | — (reserved; not currently used in existing CSS) |
| 3xl | 64px | PostGame page top padding |

Exceptions:
- Touch targets: all buttons must have `min-height: 44px` — exceeds 8-point scale to meet accessibility minimum. This is a hard floor, not an exception to the scale. Source: `pages.css`, `GameScreen.css`, `ChatPanel.css` (all primary buttons already follow this).
- Modal overlay: modal backdrop uses full viewport coverage (`position: fixed; inset: 0`), not scale-derived.
- Reconnection banner: `top: 0; left: 0; right: 0` — full-width sticky bar, zero margin.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1117` | Page background, all screen backgrounds, input backgrounds, image panel backdrop |
| Secondary (30%) | `#1a1d27` | Cards, panels, header bar, chat panel, podium cards, modal surfaces |
| Accent (10%) | `#6366f1` | (see reserved list below) |
| Destructive | `#ef4444` | Skip/pass button border+text, error messages, negative delta scores, offline banner state, spy-discovered notification |

Accent `#6366f1` reserved for:
1. Primary action buttons background (`panel-btn-primary`, `page-submit-btn`, `chat-panel__submit`, vote-yes button)
2. Focus ring on all inputs (`*:focus-visible` outline, `chat-panel__input:focus` outline)
3. Current player's hint chip border (`hint-chip--mine`)
4. Own player name in chat message list (`chat-panel__message--mine .chat-panel__message-name`)
5. Total score column in postgame table (`postgame__table-cell--total`)
6. Reconnection banner: amber state ONLY while offline (NOT `#6366f1` — see banner spec below)

Additional semantic colors in use (not accent, not destructive):
- `#22c55e` (success / green): correct guess, positive delta score, GUESS_PHASE left border, 1st-place podium top border, restarting-game result banner border, timer state >10s
- `#3b82f6` (blue): HINT_PHASE panel left border only
- `#eab308` (yellow): SCORING_PHASE panel left border, timer state ≤10s (amber)
- `#94a3b8` (slate): chat panel title, chat message sender names
- `#6b7280` (muted): all secondary/helper text, disabled-like states, border lines (`#2d3148`)
- `#2d3148`: card/input border color (all borders except chat input which uses `#4b5563`)

Timer color contract (UI-05, D-12):
- `remainingSeconds > 10` → text color `#22c55e` (green)
- `remainingSeconds <= 10` → text color `#eab308` (amber)
- `remainingSeconds <= 5` → text color `#ef4444` (red)
- CSS transition on color: `transition: color 0.3s ease`

Delta score animation color contract (UI-07, D-13):
- Positive delta (`+N`) → `#22c55e`
- Negative delta (`-N`) → `#ef4444`
- Zero delta → `#6b7280`

Reconnection banner color contract (UI-09, D-15, D-16):
- Amber state (0–3s offline): background `#92400e`, text `#fef3c7` ("Reconectando...")
- Red state (>3s offline): background `#7f1d1d`, text `#fee2e2` ("Offline")
- Banner disappears on reconnect — no color needed for connected state

## Component Inventory (New for Phase 8)

The following net-new components must be implemented. Existing components are noted as "refine only."

### New: `ReconnectionBanner.tsx` + CSS (UI-09)
- Fixed sticky top bar: `position: fixed; top: 0; left: 0; right: 0; z-index: 100`
- Height: 40px (`min-height: 40px`; padding: 8px 16px)
- Does NOT push page content down — game content must have `padding-top: 40px` when banner is visible
- Listens: `socket.on('disconnect')` → show amber; `useRef` timer → 3s → switch to red; `socket.on('connect')` → hide
- Rendered inside `GameScreen.tsx` only (D-17)
- Transition on background-color: `transition: background-color 0.5s ease`

### New: `PhaseModal.tsx` (UI-06, D-09)
- Overlay: `position: fixed; inset: 0; background: rgba(0,0,0,0.7); z-index: 50`
- Modal surface: `background-color: #1a1d27; border-radius: 12px; padding: 24px; max-width: 480px; margin: auto`
- Header: `PhaseBadge` (existing component) + phase title at 16px weight 600
- Entry animation: `@keyframes slideInFromBottom` — translateY(24px)→translateY(0), opacity 0→1, duration 250ms ease-out
- Exit: no exit animation required (modal is removed on phase change)
- Four modal variants:
  1. **HINT modal**: single text input (max 1 word, no spaces), submit button "Enviar dica", disabled after submission
  2. **GUESS modal**: target player selector (dropdown or list), text input for guess, submit button "Enviar palpite", skip button "Passar"
  3. **EXCHANGE modal — requester role**: status view (pending/accepted/waiting for partner hint), input appears only after acceptance
  4. **EXCHANGE modal — recipient role**: request notification with "Aceitar" (accent) and "Recusar" (destructive outline) buttons; after accept → hint input + "Enviar dica privada"
  5. **SPY modal**: list of active exchanges (other players' pairs), radio/click to select one, confirmation text showing -10pt risk, "Espiar" button (destructive style: `border: 1px solid #ef4444; color: #ef4444`)

### Refine: `CountdownDisplay.tsx` (UI-05, D-12)
- Add color logic based on `remainingSeconds` prop (three states above)
- Add CSS transition `transition: color 0.3s ease`
- No structural changes — existing `remainingSeconds` prop is sufficient

### New: `ScoreDeltaToast.tsx` (UI-07, D-13)
- Absolutely positioned over the scoreboard panel (not fixed to viewport)
- Animates with `@keyframes slideUpFade`: translateY(0)→translateY(-32px), opacity 1→0
- Duration: 1.5s, `animation-fill-mode: forwards`
- Removed from DOM after animation completes (`onAnimationEnd` → setState remove)
- Font: 20px weight 600 — larger than body to be visible
- Triggered per player on SCORE_UPDATED event

### Refine: `ChatPanel.tsx` (UI-10)
- Visual polish only — layout and separation already correct per Phase 7
- Ensure label reads "Chat" (heading) and input placeholder "Mensagem..." (distinct from game input)
- Submit button label: "Enviar mensagem" (not "Enviar")
- No layout changes

## Animation Keyframes (Phase 8 additions to `index.css` or `GameScreen.css`)

```css
/* Phase modal entry */
@keyframes slideInFromBottom {
  from { opacity: 0; transform: translateY(24px); }
  to   { opacity: 1; transform: translateY(0); }
}

/* Score delta float-up */
@keyframes slideUpFade {
  0%   { opacity: 1; transform: translateY(0); }
  70%  { opacity: 1; transform: translateY(-24px); }
  100% { opacity: 0; transform: translateY(-32px); }
}
```

Transition durations:
- Phase modal entry: 250ms ease-out (D-14 — "200ms–400ms Claude decides")
- Timer color change: 300ms ease (smooth without lag)
- Vote bar width + color: `transition: width 1s linear, background-color 0.5s ease` (already implemented in PostGame.css)
- Reconnection banner background: 500ms ease

## Phase-Specific Interaction States

All interactive elements must have:
1. Default state
2. Hover state (`opacity: 0.9` for primary buttons, `border-color: #6366f1` for inputs on hover)
3. Focus state (`outline: 2px solid #6366f1; outline-offset: 2px` — already declared globally in `index.css`)
4. Disabled state (opacity 0.5–0.75, `cursor: not-allowed` — patterns already established in existing CSS)
5. Loading / submitted state for form buttons: text changes to "Enviando..." + disabled during async; no spinner required

Modal backdrop click: does NOT close the modal (game actions must be intentional; phase modals are closed by phase transition only).
