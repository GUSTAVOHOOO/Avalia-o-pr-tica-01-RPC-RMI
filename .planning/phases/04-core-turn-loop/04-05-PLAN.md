## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser form inputs → socket.emit | hint_word and guess_word from input fields cross here; length limited by maxLength on input elements (defense-in-depth; server is authoritative) |
| socket.emit player identity | player_id never included in submit_hint/submit_guess emit payloads — bridge resolves from SID |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Spoofing | Frontend submit_hint / submit_guess payload | mitigate | player_id intentionally omitted from all socket.emit calls. Bridge resolves identity from SID map — cannot be spoofed by client. |
| T-4-02 | Tampering | hint input maxLength=30, guess input maxLength=50 | mitigate | Frontend maxLength provides UX guard; server-side strip()[:50] is the authoritative enforcement (Plan 03). |
| T-4-03 | Tampering | Self in target dropdown | mitigate | Frontend filters self from GuessPhasePanel target dropdown options (players.filter(p => p.player_id !== myPlayerId)). Server-side rejection is authoritative (Plan 03). |
| T-4-04 | Tampering | Image URL in OBJECT_ASSIGNED payload | accept | image_url comes from bridge/server side (_assign_images_for_turn). Frontend renders it as <img src={url}> — no arbitrary URL injection by client. Flask send_from_directory prevents path traversal on server side. |
</threat_model>
