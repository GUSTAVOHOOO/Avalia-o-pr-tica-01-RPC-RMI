## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Spoofing | submit_hint / submit_guess player_id | mitigate | Bridge resolves player_id from _sid_to_player[request.sid] — never from client payload. GameServer validates player_id is in session via _player_to_room lookup. |
| T-4-02 | Tampering | hint_word / guess_word injection | mitigate | str(hint_word).strip()[:50] and str(guess_word).strip()[:50] applied before any storage or comparison. Lowercased for comparison only — stored value retains original case (display). |
| T-4-03 | Tampering | Player guessing own object | mitigate | submit_guess() returns {"error":"cannot_guess_own_object"} when player_id == target_player_id. Frontend also hides self from dropdown (Plan 05), but server enforcement is authoritative. |
| T-4-04 | Tampering | Image path traversal | accept | _image_manifest filenames are loaded from server-controlled manifest.json; no client input selects a filename. send_from_directory path protection added in Plan 04. |
