<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Bridge Socket.IO | All client-submitted data crosses here; player_id is never accepted from payload |
| Bridge callback thread → socketio.emit | BridgeCallbackReceiver.on_* runs in Pyro5 daemon thread; socketio.emit is thread-safe in threading mode |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Spoofing | handle_submit_hint, handle_submit_guess, handle_skip_guess | mitigate | player_id resolved exclusively from _sid_to_player[request.sid] — client payload never used for identity. If SID not in map, handler returns error immediately. |
| T-4-02 | Tampering | hint_word / guess_word in bridge handlers | mitigate | Bridge passes raw strings to GameServer which applies .strip()[:50] (implemented in Plan 03). Bridge is not the enforcement layer — server is. |
| T-4-03 | Tampering | Self-target guess | mitigate | Bridge passes target_player_id to GameServer.submit_guess() which rejects self-target. Frontend additionally hides self from dropdown (Plan 05). |
| T-4-04 | Tampering | Image path traversal via /static/images/ | mitigate | send_from_directory(server/images/, filename) uses Flask's built-in path-traversal protection — relative path components (../) are rejected by Flask. Filenames originate from server-controlled manifest, not from client input at request time. |
</threat_model>
